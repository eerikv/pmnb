WATERFALL PROJECT MANAGEMENT

Basic description of Waterfall Project Management

Waterfall Project Management usually understood to be:

  • Emphasizes linear progression from the beginning to the end
  • Front-loaded methodology: relies on careful planning, detailed documentation and consecutive execution
  • Sequential development process that flows like a waterfall through all the phases of the project
  • Each phase completed before beginning the next phase

The phases of the Waterfall model :

  1. Requirements
  2. Design
  3. Implementation
  4. Testing
  5. Deployment
  6. Maintenance

Why implementing Waterfall Project Management?

  • Waterfall methodology is a straightforward, well-defined management methodology -> easy to manage
  • It is very easy to understand
  • Requirements are clearly laid out from the beginning
  • Each contributor knows what needs to be done and when
  • Helps planning the resources effectively
  • Executives and decision makers prefer to see beforehand the required resources
  • They also like it that with waterfall method it is easy to show the status of the project (clearly defined milestones)
